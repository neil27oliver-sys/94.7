94.7 Outdoor Lab App: Phone Setup & Compatibility Guide

Our app is a Progressive Web App (PWA) ‚Äì it runs right in your phone's browser (mostly Chrome on Android, Safari on iOS) without needing to download from a store. This keeps it lightweight and updatable instantly. But to get the full experience (quick launch from home screen, reliable Bluetooth for speed/cadence/power/HR sensors, accurate GPS speed/distance), your phone needs to meet some basics.

Why Chrome Must Be Your Default Browser

For the 94.7 app to work smoothly ‚Äî especially when connecting Bluetooth sensors (like your Saris speed/cadence) and installing the app to your home screen ‚Äî your phone needs Google Chrome set as the default browser.

Why Chrome?
- Most reliable Bluetooth Web API support
- Best Progressive Web App (PWA) installation experience
- Other browsers (Samsung Internet, Firefox, Edge) often block or limit Bluetooth pairing in web apps

How to set Chrome as default (Android):

  1. Open your phone's Settings app
  2. Go to Apps (or Apps & notifications)
  3. Tap Default apps (sometimes under Advanced)
  4. Tap Browser app (or Opening links / Browser)
  5. Select Google Chrome

iOS note: iOS doesn't have a true "default browser" setting like Android, but make sure you're using Safari or Chrome when accessing the app link. Bluetooth pairing works best in Chrome on iOS too.

Do this before trying to connect sensors or install the app to avoid connection issues!

1. Adding the App Icon to Your Home Screen (Make It Feel Like a Real App)

This creates a shortcut/icon on your home screen that opens the app full-screen (no browser bars), just like a downloaded app.

On Android (Chrome browser ‚Äì recommended):

  1. Open Chrome and go to the app URL (e.g., your hosted link for outdoor.html or the main page).
  2. Tap the 3-dot menu (top-right).
  3. Select Add to home screen (or "Install app" on newer versions ‚Äì it might pop up automatically after a few visits).
  4. Give it a name like "94.7 Outdoor Lab" and tap Add.
  5. The icon appears on your home screen ‚Äì drag it where you want.
  6. Bonus: It should launch full-screen without the address bar.

On iPhone/iPad (Safari browser):

  1. Open Safari and load the app URL.
  2. Tap the Share icon (square with up-arrow, bottom bar).
  3. Scroll down and tap Add to Home Screen.
  4. Name it (e.g., "94.7 Lab") and tap Add (top-right).
  5. Icon added to home screen ‚Äì tap to launch full-screen.

Troubleshooting icon issues:

  • If no "Add to home screen" option shows: Visit the site a few times, engage with it (e.g., start a ride simulation), then try again ‚Äì browsers sometimes need "engagement" to offer PWA install.
  • Wrong/old icon? Clear browser cache/site data or wait for server update (if we add a manifest.json with custom icon later).
  • Icon missing on older phones? Some very old Android/iOS versions don't support PWAs well ‚Äì see compatibility below.

2. Bluetooth Sensor Connection ‚Äì Minimum Phone Requirements

The app uses Web Bluetooth to connect to cycling sensors (speed/cadence, power, heart rate). This only works on supported browsers/versions.

Android:

  • Minimum: Android 6.0 (Marshmallow) or newer + Chrome browser.
  • Works best: Android 10+ (better permissions handling).
  • Won't work on: Android 5.1 or older ‚Äì Web Bluetooth isn't supported at all.

Common fixes for connection fails:

  • Bluetooth ON + Location services ON (required for scanning BLE devices on Android 6‚Äì11; newer versions are smarter).
  • No other app connected to the sensor (e.g., close Zwift/Garmin Connect).
  • Grant permissions when prompted (nearby devices + location).
  • Restart phone/Chrome if flaky.

iOS (iPhone/iPad):

  • No full Web Bluetooth support in Safari (as of 2025/2026) ‚Äì can't reliably connect to most cycling sensors via browser.
  • Workaround: Use a third-party browser if available, but generally, iOS users may need native apps (like Strava, Garmin Connect) for sensor data instead of our web version.
  • GPS still works fine on iOS 13+.

3. GPS Tracking ‚Äì Minimum Requirements

We use the browser's Geolocation API for live speed, distance, and mapping.

Android:

  • Works from Android 4.4+ in Chrome, but high-accuracy GPS (using actual GPS chip) needs Android 6+ with Location set to High accuracy mode.
  • Old phones (Android 5 or older): GPS may be slow, inaccurate, or battery-draining ‚Äì or not work at all in browser.

iOS:

  • Works well from iOS 11+ in Safari (high-accuracy with permission).

Tips for best GPS:

  • Set phone location mode to High accuracy (GPS + Wi-Fi + mobile).
  • Grant location permission when prompted.
  • Test outdoors with clear sky ‚Äì indoor/urban canyons can be spotty.
